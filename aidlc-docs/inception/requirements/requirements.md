# 要件定義書

## インテント分析

- **ユーザーリクエスト**: RSSフィードURLからポッドキャスト等のコンテンツ一覧を表示し、タイトル・作成日・オーディオの長さ・概要を確認したうえで、オーディオファイルをダウンロードできるGUIツール
- **リクエストタイプ**: 新規プロジェクト（Greenfield）
- **スコープ**: 単一デスクトップアプリケーション
- **複雑度**: 中程度（GUI + RSS解析 + ファイルダウンロード + 設定管理）

---

## 技術スタック

| 項目 | 選定内容 | 理由 |
|---|---|---|
| 言語 | Python 3.11+ | クロスプラットフォーム対応、RSS/HTTPライブラリが豊富 |
| GUIフレームワーク | PySide6（Qt for Python） | リッチなUI、クロスプラットフォーム対応優秀、LGPLライセンス |
| RSS解析 | `feedparser` | Python標準的なRSS/Atomパーサー |
| HTTPダウンロード | `requests` または `httpx` | 進捗表示対応、信頼性が高い |
| 設定保存 | JSON / `QSettings` | OS標準の設定ストレージ |
| パッケージ管理 | `pip` + `pyproject.toml` | 標準的なPythonパッケージ管理 |

## 機能要件

### FR-01: RSSフィード管理
- ユーザーは複数のRSSフィードURLを登録・編集・削除できる
- フィードURLは設定ファイルに永続化される
- フィードには任意の名前（ラベル）を付けられる

### FR-02: コンテンツ一覧表示
- 選択したRSSフィードのエピソード一覧を表示する
- 一覧に表示する項目：
  - **タイトル**
  - **公開日時**（読みやすい形式：例 `2024年6月21日 10:30`）
  - **オーディオの長さ**（`HH時間MM分SS秒` 形式、または `MM分SS秒`）
  - **概要**（description / summary）
  - **ダウンロード状態**（未ダウンロード / ダウンロード済み）

### FR-03: オーディオ長さの取得
- RSSフィードのメタデータ（`<itunes:duration>`、`<enclosure>`等）から取得する
- 秒数・`MM:SS`・`HH:MM:SS` など複数フォーマットを正規化して表示する

### FR-04: エピソードのダウンロード
- 一覧から複数エピソードを選択して一括ダウンロードできる
- ダウンロード先フォルダは設定画面で事前に指定する
- ダウンロード済みのファイルがすでに存在する場合は上書きするか確認する

### FR-05: ダウンロード進捗表示
- ダウンロード中はプログレスバーで進捗率（%）を表示する
- 複数ファイルの場合は全体の進捗と個別の進捗を表示する
- ダウンロード完了・エラー時に通知する

### FR-06: 設定管理
- 設定画面から以下を変更できる：
  - ダウンロード先フォルダ（フォルダ選択ダイアログ付き）
  - 登録済みRSSフィードURL一覧の管理（追加・編集・削除）
- 設定はアプリ終了時に自動保存される

## 非機能要件

### NFR-01: クロスプラットフォーム対応
- Windows 10以降、macOS 12以降、主要Linux ディストリビューション（Ubuntu 22.04等）で動作する

### NFR-02: 起動・応答性
- アプリ起動時間：5秒以内
- RSS取得はバックグラウンドスレッドで実行し、UIをブロックしない
- ダウンロードはバックグラウンドで実行し、UIは操作可能な状態を維持する

### NFR-03: エラーハンドリング
- ネットワーク接続エラー時はユーザーにわかりやすいエラーメッセージを表示する
- 不正なRSSフィードURLの場合は入力時にバリデーションエラーを表示する
- ダウンロードの部分的な失敗（一部ファイルのみ失敗）を適切に報告する

### NFR-04: ユーザービリティ
- 日本語UIをサポートする（主要ラベル・メッセージは日本語）
- ウィンドウサイズはリサイズ可能
- 前回のウィンドウ位置・サイズを記憶する

### NFR-05: テスト
- 通常のユニットテストを実装する（PBTは採用しない）
- 主なテスト対象：RSS解析ロジック、durationパース処理、設定の読み書き

## エクステンション設定

| エクステンション | 状態 | 決定理由 |
|---|---|---|
| セキュリティベースライン | 無効（スキップ） | PoC・個人ツール用途のため |
| レジリエンシーベースライン | 無効（スキップ） | デスクトップ個人ツールには過剰 |
| プロパティベーステスト | 無効（スキップ） | 通常のユニットテストで十分 |

## 制約・前提条件

- オーディオファイルの形式はRSSフィードで配信されるもの（MP3、M4A等）をそのままダウンロードする（変換は行わない）
- RSSフィードはHTTP/HTTPSでアクセス可能なものを対象とする
- 認証が必要なRSSフィード（Basic認証等）は初期バージョンでは対象外
- ダウンロードの同時並行数は設定可能（デフォルト：2）

---

## 成功基準

1. 任意のポッドキャストRSSフィードURLを登録してエピソード一覧を表示できる
2. 一覧からエピソードを複数選択して一括ダウンロードできる
3. ダウンロード中にプログレスバーで進捗を確認できる
4. Windows・macOS・Linuxの3プラットフォームで動作する
