# 非機能要件 — Unit 1: core

## NFR-01: パフォーマンス

| 要件 | 値 | 備考 |
|---|---|---|
| RSSフィード取得タイムアウト | 30秒 | 低速回線を考慮 |
| ダウンロードチャンクサイズ | 8,192 bytes | メモリ消費とスループットのバランス |
| エピソード表示上限 | なし | フィード全エピソードを表示 |
| 同時ダウンロード数 | 5（固定） | `MAX_CONCURRENT_DOWNLOADS = 5` |
| UIブロック禁止 | RSS取得・ダウンロードは必ずバックグラウンドスレッド | メインスレッドをブロックしない |

---

## NFR-02: 信頼性・エラーハンドリング

| 要件 | 内容 |
|---|---|
| ネットワークエラー | リトライなし。即座にエラーメッセージを表示 |
| ダウンロード中断 | 中断ファイルを削除。ステータスをリセット |
| 設定ファイル破損 | デフォルト設定で起動し、ユーザーに通知 |
| RSSパースエラー | エラーを表示し、既存キャッシュを維持 |
| ダウンロード先フォルダ不在 | 自動作成を試みる。失敗時はエラー通知 |

---

## NFR-03: 保守性・ログ

| 要件 | 内容 |
|---|---|
| ログ出力先 | `{config_dir}/app.log` |
| ログフォーマット | `%(asctime)s [%(levelname)s] %(name)s: %(message)s` |
| ログレベル | 通常運用: `INFO`。デバッグ時: `DEBUG`（環境変数 `PODCAST_DL_DEBUG=1` で切り替え） |
| ログローテーション | 最大 5MB × 3世代（`RotatingFileHandler` 使用） |
| ログ対象イベント | フィード取得開始/完了/エラー、ダウンロード開始/完了/エラー、設定読み書き |

---

## NFR-04: 移植性・環境要件

| 要件 | 内容 |
|---|---|
| 最低Pythonバージョン | Python 3.11以上 |
| 対応OS | Windows 10以降、macOS 12以降、Ubuntu 22.04以降（主要Linux） |
| 設定ディレクトリ | OS標準パス（`platformdirs` ライブラリで解決） |
| 外部コマンド依存 | なし（すべてPythonライブラリで完結） |

---

## NFR-05: テスト容易性

| 要件 | 内容 |
|---|---|
| core は GUI 非依存 | PySide6 に依存しない。単体でテスト可能 |
| `duration_parser` は純粋関数 | 副作用なし。入出力のみでテスト可能 |
| `rss_parser` はモック対応 | HTTPクライアントを DI または モンキーパッチでモック可能 |
| `downloader` はコールバックテスト | コールバック関数をモックして進捗・完了・エラーを検証 |
| テストフレームワーク | `pytest` |
