# ビジネスルール — Unit 1: core

## BR-01: RSSフィード登録バリデーション

| ルール | 内容 |
|---|---|
| BR-01-1 | URL は `http://` または `https://` で始まること |
| BR-01-2 | URL は空文字不可 |
| BR-01-3 | 同一 URL のフィードはすでに登録されていた場合は重複登録エラー |
| BR-01-4 | ラベルは空文字不可。未入力時は URL のホスト名部分をデフォルトラベルとする |

---

## BR-02: RSSパース・エピソードフィルタリング

| ルール | 内容 |
|---|---|
| BR-02-1 | `audio_url`（enclosure の url 属性）が欠損または空のエピソードはスキップ（Episodeを生成しない） |
| BR-02-2 | `title` が欠損した場合は `"（タイトルなし）"` を使用する |
| BR-02-3 | `published` が欠損した場合は `None`（一覧では `"日時不明"` と表示） |
| BR-02-4 | `description` が欠損した場合は空文字 `""` を使用する |
| BR-02-5 | 同一フィード内で `guid` が重複するエピソードは後出しのものを無視する |
| BR-02-6 | フィード取得後、既存キャッシュとのマージ時は `guid` をキーにして重複排除する |

---

## BR-03: Durationパース規則

| ルール | 内容 |
|---|---|
| BR-03-1 | 対応フォーマット: 秒数のみ（`"3600"`）、`"MM:SS"`、`"HH:MM:SS"` |
| BR-03-2 | いずれのフォーマットにも一致しない場合は `None`（表示: `"不明"`） |
| BR-03-3 | `MM:SS` で分が60以上の場合は時間に繰り上げる（例: `"90:00"` → 5400秒 → `"1時間30分00秒"`） |
| BR-03-4 | 秒数が0未満の場合は `None` とみなす |
| BR-03-5 | 表示形式: 1時間以上は `HH時間MM分SS秒`、1時間未満は `MM分SS秒`、1分未満は `SS秒` |

---

## BR-04: ファイル命名規則

| ルール | 内容 |
|---|---|
| BR-04-1 | ファイル名フォーマット: `{フィードラベル}_{YYYYMMDD}_{タイトル}.{拡張子}` |
| BR-04-2 | 拡張子は音声URLから取得する（例: `.mp3`, `.m4a`）。取得できない場合は `.mp3` |
| BR-04-3 | サニタイズ: ファイル名に使用できない文字（`/ \ : * ? " < > |`）はアンダースコア `_` に置換 |
| BR-04-4 | ファイル名の最大長は255バイト（超過時はタイトル部分を切り詰め） |
| BR-04-5 | `published` が `None` の場合は日付部分を `"00000000"` とする |

**例**:
```
フィードラベル: "Tech Podcast"
公開日: 2024-06-21
タイトル: "Episode 42: AI の話"
拡張子: .mp3

→ "Tech Podcast_20240621_Episode 42_ AI の話.mp3"
   ↓ サニタイズ後
   "Tech Podcast_20240621_Episode 42_ AI の話.mp3"
```

## BR-05: ダウンロード済み重複チェック

| ルール | 内容 |
|---|---|
| BR-05-1 | ダウンロード開始前に、予定ファイルパスのファイルが実際に存在するか確認する |
| BR-05-2 | ファイルが存在する場合は `DownloadStatus.DOWNLOADED` として扱い、スキップする |
| BR-05-3 | ファイルが存在しない場合はキャッシュのステータスにかかわらずダウンロードを実行する |
| BR-05-4 | スキップしたエピソードはユーザーに通知する（ダウンロード済みとして表示） |

---

## BR-06: ダウンロード中断時の処理

| ルール | 内容 |
|---|---|
| BR-06-1 | ダウンロード中断時（エラー・キャンセル問わず）は中断された一時ファイルを削除する |
| BR-06-2 | エピソードのステータスを `ERROR`（エラー時）または `NOT_DOWNLOADED`（キャンセル時）に更新する |
| BR-06-3 | エラーメッセージはユーザーが理解できる日本語で表示する |

---

## BR-07: エラーハンドリング方針

| ルール | 内容 |
|---|---|
| BR-07-1 | ネットワークエラー（RSS取得・ダウンロード）は即座にエラーとしてユーザーに通知する（リトライなし） |
| BR-07-2 | 不正なRSSフォーマット時は `RSSParseError` を送出し、エラーメッセージを表示する |
| BR-07-3 | 設定ファイルが破損している場合はデフォルト設定で起動し、ユーザーに通知する |
| BR-07-4 | ダウンロード先フォルダが存在しない場合は自動作成を試みる。失敗時はエラー通知 |

---

## BR-08: 同時ダウンロード数制御

| ルール | 内容 |
|---|---|
| BR-08-1 | 同時実行ダウンロード数は最大 5（固定値、ユーザー変更不可） |
| BR-08-2 | 上限に達した場合、残りのエピソードはキューに追加し、スロットが空き次第開始する |

---

## BR-09: キャッシュ更新ルール

| ルール | 内容 |
|---|---|
| BR-09-1 | アプリ起動時にすべての登録済みフィードをバックグラウンドで自動取得する |
| BR-09-2 | ユーザーが手動で「更新」ボタンを押した場合も即時取得する |
| BR-09-3 | 取得したエピソードは既存キャッシュと `guid` でマージする（既存エピソードのステータスを保持） |
| BR-09-4 | キャッシュ取得中にエラーが発生した場合は既存キャッシュをそのまま表示し、エラーをユーザーに通知する |
