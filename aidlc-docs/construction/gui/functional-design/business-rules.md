# Business Rules — Unit 2: gui

## BR-GUI-01: フィード追加バリデーション

| ルール | 内容 |
|---|---|
| BR-GUI-01-1 | URL は `http://` または `https://` で始まること。違反時はインラインエラーラベルを表示（ダイアログは閉じない） |
| BR-GUI-01-2 | 同一 URL がすでに登録されている場合は重複エラーメッセージを表示 |
| BR-GUI-01-3 | ラベルが空の場合は URL のホスト名を自動設定 |
| BR-GUI-01-4 | OK ボタン押下時に実際に RSS 取得を試みて有効なフィードか確認する。失敗時は `QMessageBox.critical` を表示 |

---

## BR-GUI-02: フィード削除

| ルール | 内容 |
|---|---|
| BR-GUI-02-1 | フィード削除前に `QMessageBox.question` で確認ダイアログを表示する |
| BR-GUI-02-2 | 確認後、設定からフィードを削除し、対応するエピソードキャッシュファイルを削除する |
| BR-GUI-02-3 | 削除されたフィードが選択中の場合はエピソード一覧をクリアする |

---

## BR-GUI-03: エピソード選択・ダウンロード

| ルール | 内容 |
|---|---|
| BR-GUI-03-1 | チェックボックスと行選択は双方向に同期する（行クリック→チェックボックス更新、チェックボックス操作→行選択状態更新） |
| BR-GUI-03-2 | ダウンロードボタンは1件以上チェックされているときのみ有効 |
| BR-GUI-03-3 | ダウンロード中（`DOWNLOADING`）のエピソードは再選択してもダウンロードキューに追加しない |
| BR-GUI-03-4 | ダウンロード済み（`DOWNLOADED`）のエピソードを選択してダウンロードボタンを押した場合は確認ダイアログを表示する（スキップまたは上書き） |

---

## BR-GUI-04: エラー通知

| ルール | 内容 |
|---|---|
| BR-GUI-04-1 | ネットワークエラー（RSS取得・ダウンロード失敗）は `QMessageBox.critical` で表示する |
| BR-GUI-04-2 | エラーダイアログには「エラー内容」と「対処方法のヒント」を含める |
| BR-GUI-04-3 | バックグラウンドスレッドからのエラー通知は必ずメインスレッドで受け取ってからダイアログを表示する |

---

## BR-GUI-05: ウィンドウ状態の永続化

| ルール | 内容 |
|---|---|
| BR-GUI-05-1 | アプリ終了時にウィンドウの位置・サイズを `AppSettings.window_geometry` に保存する |
| BR-GUI-05-2 | 起動時に保存済みジオメトリがあれば復元する。ない場合はデフォルトサイズ（1024×768）で起動する |
| BR-GUI-05-3 | 復元後にウィンドウがスクリーン外にはみ出す場合はデフォルト位置にリセットする |

---

## BR-GUI-06: ダウンロード進捗表示

| ルール | 内容 |
|---|---|
| BR-GUI-06-1 | ダウンロード開始時に `DownloadPanel` を表示する |
| BR-GUI-06-2 | `content-length` ヘッダーがない場合はプログレスバーを不定モード（`setRange(0, 0)`）にする |
| BR-GUI-06-3 | 全件完了（成功・エラー問わず）後は `DownloadPanel` を非表示に戻す |
| BR-GUI-06-4 | キャンセルボタン押下時は `DownloadManager.cancel_all()` を呼び出す |

---

## BR-GUI-07: スプリッターとレイアウト

| ルール | 内容 |
|---|---|
| BR-GUI-07-1 | 左ペイン（FeedPanel）の最小幅は 180px、初期幅は 220px |
| BR-GUI-07-2 | 右ペイン内の概要ペイン（DescriptionPane）は垂直スプリッターで高さ調整可能 |
| BR-GUI-07-3 | スプリッター位置はウィンドウジオメトリと共に保存・復元する |
